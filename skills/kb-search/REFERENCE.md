`fileskb` MCP server（或同等功能的工具）提供以下能力來從 markdown 文件目錄中擷取相關資訊：

* **list_contents** — 瀏覽目錄樹，列出指定路徑下的內容（檔案或子目錄）。對應抽象能力「列出目錄」。
* **read_file** — 讀取知識庫中的檔案完整內容，從中擷取並摘要相關資訊。對應抽象能力「讀檔」。
* **search_content** — 搜尋知識庫中的關鍵字，回傳符合的檔名、行號與片段。可用來快速定位而不需逐一列出和讀取所有檔案。

使用方式：先用簡單的關鍵字查詢找到相關文件，再讀取檔案取得詳細內容。若一次查詢無結果，嘗試不同關鍵字組合。

## 若沒有 `fileskb` 這類專屬 MCP

改用 `SKILL.md` 前置條件定義的「列出目錄」「讀檔」這兩個抽象能力（此環境目前對應到 `Filesystem:list_directory`、`Filesystem:read_text_file`——這兩個是此環境專屬的 Filesystem MCP 工具名稱，換到別的 MCP 只需要換掉這兩個名稱）：

* 「列出目錄」取代 `list_contents`。
* 「讀檔」取代 `read_file`。
* 沒有對應 `search_content` 的關鍵字搜尋能力時，改成先「列出目錄」取得檔案清單，再逐一「讀檔」後自行比對關鍵字；檔案數量多時，先跟使用者確認是否要縮小搜尋範圍（例如指定子目錄），避免一次讀取整個知識庫造成大量工具呼叫。

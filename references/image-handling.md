# 圖片歸檔規則（個人擴充，非 OKF 官方規範）

OKF 規範本身沒有定義任何圖片或附件的處理方式，這一段是使用者自訂、疊加在 OKF 之上的個人慣例。使用時要跟使用者說清楚這是自訂規則，不是 OKF 標準行為，避免日後誤以為所有 OKF consumer 都認得這個結構。

> 本文件用「建立目錄」「搬移檔案」等抽象能力描述步驟，對應到具體工具的名稱見 `SKILL.md` 前置條件的對應表；那張表列的是此環境目前連接的 Filesystem MCP 專屬工具名稱，換到別的 MCP 只需要改那張表，這裡的規則不用跟著改。

## 基本規則

當這次輸入的來源裡含有圖片檔案（螢幕截圖、附圖、掃描檔、流程圖等）：

1. **記錄原始檔名**：不論最後放在哪裡，都要記下每張圖片的原始檔名，這個檔名之後會寫進對應 concept 文件的 frontmatter。
2. **決定所屬分類**：圖片歸屬到「引用它的 concept 文件」所在的分類子目錄，不是歸屬到 bundle 根目錄。一張圖片如果被多份文件引用，圖片本身只存放一份，放在第一份引用它的文件所在分類底下，其餘文件用相對路徑往回指（例如 `../other-category/img/xxx.png`），並在對話中提醒使用者這種跨分類引用的情況。
3. **建立 img 資料夾**：路徑固定是 `<bundle 根目錄>\<分類子目錄>\img`。用「建立目錄」建立，這個能力若資料夾已存在會直接成功，不用先檢查存不存在。
4. **搬移檔案**：用「搬移檔案」把圖片從原始路徑搬到 `<分類子目錄>\img\<原始檔名>`。

   - 這是「搬移」不是「複製」——這個環境的檔案系統工具通常只提供搬移，沒有單機內複製的工具。搬移後原始位置就沒有這個檔案了，執行前要讓使用者知道這一點，尤其原始檔案如果還有其他用途。
   - 若使用者要求保留原始位置的檔案，改用讀取＋在 Claude 沙盒暫存再寫出的方式處理（即先讀入，再用電腦工具寫到暫存路徑，最後仍須提醒使用者：目前工具沒有辦法在使用者電腦上直接產生第二份複本，只能由使用者自己手動複製）。

5. **檔名衝突處理**：若目標 `img` 資料夾底下已經有同名檔案，不要覆蓋，改成 `<原始檔名前綴>-2.<副檔名>`、`-3` 依此類推，並在完成後的摘要裡明確告知使用者哪些檔名被改過。

## 在 concept 文件裡的記錄方式

### frontmatter 擴充欄位

```yaml
---
type: Playbook
title: VPN 連線疑難排解
description: VPN 連線失敗時的排查步驟。
tags: [it-support, vpn]
timestamp: 2026-07-09T10:00:00Z
images: [vpn-error-screenshot.png, vpn-settings-page.png]
---
```

`images` 欄位列出這份文件用到的所有圖片原始檔名（陣列），方便日後用程式掃描 frontmatter 統計某份文件用了哪些圖，不需要另外解析 body 內的 markdown image 語法。

### body 內嵌入方式

在實際引用圖片的地方，用相對於該 concept 文件的路徑插入：

```markdown
出現以下畫面代表 VPN 憑證過期：

![VPN 錯誤畫面](img/vpn-error-screenshot.png)
```

若這份文件本身放在分類子目錄底下（例如 `it-support/vpn-troubleshooting.md`），圖片路徑就是 `img/xxx.png`（相對於同一層的 img 資料夾），不需要加上分類名稱前綴。

## 邊界情況

- **來源本身就是圖片、沒有任何文字說明**：仍然要幫它建一份 concept 文件（不能只把圖丟進 img 資料夾就結束），文件內容至少要有 frontmatter（`type` 用類似 `Reference` 或依內容判斷更精確的類型）與一段簡短描述這張圖在講什麼，再用上面的方式嵌入圖片。單純把檔案搬進 img 資料夾、卻沒有任何 concept 文件指向它，會變成「孤兒圖片」，之後不容易被知道它的用途，要避免這種情況。
- **無法判斷圖片該歸哪個分類**：不要硬塞進第一個看到的分類。先詢問使用者這張圖片屬於哪個主題，或者先歸到最相關的既有分類並在完成後的摘要裡明確標註「這張圖片的分類是我推測的，如果不對請告訴我」。
- **非圖片的其他附件（例如 log 檔、設定檔匯出）**：這份規則只涵蓋圖片。其他附件類型如果使用者也想歸檔，先確認要不要延伸同樣的 `img` 慣例（例如另開一個 `files` 或 `attachments` 資料夾），不要自行假設沿用 `img` 這個資料夾名稱來放非圖片內容。
